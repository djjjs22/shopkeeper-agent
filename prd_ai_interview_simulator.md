## 1. 产品概述

### 1.1 产品定位

> 上传简历 + 选择目标岗位 → AI 自动生成个性化面试题 → 模拟真实面试 → 即时评分 + 改进建议

### 1.2 一句话价值主张

**让你的面试准备不再是"刷题"，而是"排练"——AI 面试官根据你的简历出题，考你最可能被问到的问题。**

### 1.3 商业模式

| 套餐 | 价格 | 内容 |
|------|------|------|
| 免费版 | ¥0 | 每天 3 道题 + 基础评分 |
| Pro 月付 | ¥9/月 | 无限题目 + 个性化出题 + 详细评分报告 + 题目历史 |
| Pro 年付 | ¥69/年（¥5.75/月） | 同上 + 优先新功能 |
| 单次购买 | ¥3/场 | 模拟一场完整面试（5-8 题），适合一次性用户 |

### 1.4 成功指标（MVP 上线后 1 个月）

| 指标 | 目标值 |
|------|--------|
| 注册用户数 | ≥ 500 |
| Pro 付费转化率 | ≥ 5% |
| 人均单次题目数 | ≥ 8 题 |
| 次日留存率 | ≥ 40% |
| NPS（净推荐值） | ≥ 30 |

## 5. 功能详细说明

### 5.1 简历上传与解析

**流程**：
```
用户进入首页 → 点击"上传简历" → 选择文件 →
前端显示上传进度 → 后端解析文件内容 →
前端展示解析结果（姓名/学校/经历等） → 用户确认或手动修改
```

**支持格式**：PDF、Word（.doc/.docx）、图片（PNG/JPG）

**解析要求**：
- 提取：姓名、邮箱、学校、专业、工作/实习经历（公司+职位+时间+描述）、项目经历、技能标签
- 输出：结构化 JSON（见数据模型 6.1）
- 容错：解析失败时允许用户手动粘贴纯文本简历

**验收标准**：
- [ ] PDF 解析准确率 ≥ 85%（中英文简历）
- [ ] Word 文件解析成功，不乱码
- [ ] 图片简历调用 OCR 后正确识别中文
- [ ] 上传 10MB 以内的文件不超时

---

### 5.2 岗位选择

**流程**：
```
展示岗位输入框（支持下拉 + 搜索 + 自由输入）
  ├── 预设热门岗位（后端开发 / 前端开发 / 产品经理 / 数据分析 / AI工程师）
  └── 自由输入岗位名称
用户选择/输入 → 显示岗位标签
可选：上传目标 JD 文本（粘贴即可）
```

**岗位分类（预设）**：

| 大类 | 子类举例 |
|------|---------|
| 技术 | 后端、前端、算法、数据工程、AI应用、测试、运维 |
| 产品 | 产品经理、AI产品、增长产品、策略产品 |
| 设计 | UI/UX、视觉设计、交互设计 |
| 运营 | 用户运营、内容运营、电商运营 |
| 市场 | 品牌营销、数字营销、商务拓展 |
| 其他 | 自由输入 |

**验收标准**：
- [ ] 预设岗位 ≥ 20 个
- [ ] 自由输入岗位可以正常出题
- [ ] 如果用户上传了 JD 文本，AI 出题应参考 JD 中的关键词

### 5.3 AI 出题引擎

**这是产品的核心模块，需要重点设计。**

**出题规则**：

```
输入：简历结构化数据 + 岗位类型 + （可选：JD文本）
  ↓
Prompt 模板要求 AI：
  1. 基于简历中的项目经历，生成至少 3 道"项目深挖题"
     - 例：简历里写了"使用 Redis 做缓存"，出题"你们的缓存策略是怎么设计的？"
  2. 基于岗位类型，生成至少 2 道"岗位通识题"
     - 例：后端岗位 → "讲讲你对微服务的理解"
  3. 如果有 JD，额外生成 1-2 道"JD 对标题"
     - 例：JD 要求"熟悉 LangGraph" → "LangGraph 和 LangChain Agent 有什么区别？"
  4. 所有题目应该是开放性问题，不是选择题
  5. 每道题附带考察点和期望回答要点（给评分系统用）
```

**输出格式**（每道题）：

```json
{
  "question": "你在项目中使用了 Redis 做缓存，请问你们的缓存策略是什么？穿透和雪崩怎么处理的？",
  "category": "project_deep_dive",
  "reference": "简历中提到的 Redis 缓存项目",
  "scoring_points": [
    "提到了缓存策略（如旁路缓存/读写穿透）",
    "正确解释了缓存穿透和雪崩的区别",
    "给出了具体的解决方案（布隆过滤器/过期时间随机化）",
    "结合了实际项目经验而非纯理论"
  ]
}
```

**验收标准**：
- [ ] 每场面试生成 5-8 道题（可配置）
- [ ] 生成时间 < 30 秒
- [ ] 至少 60% 的题目与简历内容直接相关（非泛泛通用题）
- [ ] 评分要点字段非空，可作为后续 AI 评分的参考

---

### 5.4 答题流程

**流程**：
```
① 展示题目（卡片形式）
② 用户在大文本框输入回答
③ 点击"提交"（或下一题按钮）
④ AI 即时评分 + 展示改进建议
⑤ 点击"下一题"继续
⑥ 全部答完后 → 综合报告页
```

**可选：倒计时模式（Should）**：
- 每道题可设 2-3 分钟倒计时
- 用户可开关此模式
- 倒计时结束自动提交当前回答

**验收标准**：
- [ ] 答题文本框支持 ≥ 2000 字
- [ ] 提交后评分反馈 < 10 秒
- [ ] 倒计时模式（如开发）倒计时准确，超时自动提交

---

### 5.5 AI 评分与反馈

**评分维度**：

| 维度 | 权重 | 说明 |
|------|------|------|
| 内容完整性 | 30% | 是否覆盖了评分要点 |
| 逻辑清晰度 | 25% | 回答结构是否清晰（STAR 法则） |
| 专业深度 | 25% | 是否有技术细节和量化数据 |
| 表达能力 | 20% | 语言是否流畅、简洁 |

**反馈格式**（每道题）：

```json
{
  "score": 7,
  "strength": "你提到了具体的缓存策略（旁路缓存），并且结合了实际项目场景，体现了实战经验。",
  "improvement": "缓存雪崩的解决方案中，你提到了'加锁'但没有解释如何避免锁竞争，建议补充'互斥锁+逻辑过期'的双重方案。",
  "model_answer_highlights": [
    "旁路缓存 + 逻辑过期",
    "布隆过滤器防穿透",
    "互斥锁 + 随机过期防雪崩"
  ]
}
```

**综合报告**（全场结束后）：

```json
{
  "total_score": 6.5,
  "dimensions": {
    "内容完整性": 7,
    "逻辑清晰度": 6,
    "专业深度": 5,
    "表达能力": 8
  },
  "strongest_areas": ["表达能力", "内容完整性"],
  "weakest_areas": ["专业深度"],
  "overall_feedback": "你的表达流畅、逻辑清晰，能结合项目经验回答问题。但技术细节上偏表面，建议在回答中多用量化数据和具体配置参数来体现深度。",
  "questions_review": [
    {
      "question": "...",
      "your_answer": "...",
      "score": 7,
      "key_miss": "缺少高并发下的实测数据"
    }
  ]
}
```

**验收标准**：
- [ ] 评分结果与人工评估偏差 ≤ 1.5 分（抽样 50 道题验证）
- [ ] 改进建议有**具体可操作的指导**（不是泛泛的"建议加强练习"）
- [ ] 综合报告在最后一题提交后 < 15 秒内生成

## 7. 接口定义（API）

### 7.1 接口总览

### 7.2 核心接口详情

#### POST `/api/interview/start`


## 9. 技术选型建议

### 9.1 推荐技术栈

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | React 19 + Vite + Tailwind CSS | 组件化开发快，部署简单 |
| 后端 | FastAPI（Python） | 异步支持好，与 LLM 集成方便 |
| 数据库 | PostgreSQL | 结构化数据 + JSON 字段支持 |
| 缓存 | Redis | 用户会话 + 出题结果缓存 |
| LLM | GLM-4 / DeepSeek / Qwen | 中文效果好，成本低 |
| 向量库 | Qdrant / pgvector | 面经库检索（V2 功能） |
| 文件存储 | 阿里云 OSS / 腾讯云 COS | 简历文件存储 |
| 支付 | 微信支付 JSAPI | 国内用户首选 |
| 邮件 | Resend / SendGrid | 密码重置、续费提醒 |
| 部署 | Docker + 阿里云 ECS / Railway | 低成本快速上线 |
| 监控 | Sentry（错误追踪）+ Plausible（访问统计） | 轻量够用 |

### 9.2 非功能需求

| 项目 | 要求 |
|------|------|
| 出题速度 | AI 生成 8 道题 ≤ 30 秒 |
| 评分速度 | 单题评分 ≤ 10 秒 |
| 并发支持 | MVP 阶段支持 50 并发即可 |
| 数据安全 | 简历内容加密存储，API 全量 HTTPS |
| 部署 | Docker 化，支持一键部署 |

### 12.2 AI 出题 Prompt 模板（参考）

```
【角色】你是一个资深面试官，擅长根据候选人简历进行针对性提问。

【候选人的简历】
{resume_text}

【目标岗位】
{job_title}

【目标 JD】（如有）
{jd_text}

【任务】
1. 根据简历中的项目经历，生成 3-5 道"项目深挖题"
   - 每道题必须引用简历中的具体技术/项目
   - 问题应该是开放性的，不是选择题
2. 根据目标岗位，生成 2-3 道"岗位通识题"
   - 聚焦该岗位的核心能力考察
3. 如果提供了 JD，额外生成 1-2 道"JD 对标题"
4. 每道题附带 scoring_points（3-5 个评分要点）

【输出格式】
返回 JSON 数组，每道题格式：
{
  "question": "题目内容",
  "category": "project_deep_dive | general_knowledge | jd_aligned",
  "reference": "简历中对应的描述（如有）",
  "scoring_points": ["要点1", "要点2", "要点3"]
}
```
