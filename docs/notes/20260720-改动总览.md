## 部署须知（新增环境变量）

生产环境推荐显式配置（不设则用兜底，但兜底不安全）：

```bash
# .env 或 k8s secret
QUERY_API_KEY=<random-string>           # 启用 /api/query 鉴权（不设则不鉴权）
ALLOWED_ORIGINS=https://your-domain.com  # CORS 白名单（逗号分隔，不设则只允许 localhost）
SESSION_SECRET=<random-32-bytes>        # session_id HMAC 签名密钥（不设则进程级随机，重启失效）
TRUSTED_PROXIES=10.0.0.0/8              # 信任的反代 IP（不设则只信任 127.0.0.1）
```

前端：

```bash
# .env.production
VITE_QUERY_API_KEY=<same-as-backend>    # 配合后端 QUERY_API_KEY
```

## 复盘：5 个值得记的教训

### 1. 默认参数只在 docstring 承诺、函数体没实现 = 埋坑

**场景**：`SQLSafetyValidator.validate(sql, allowed_tables=None)` 的 docstring 写"不传则用 ALLOWED_TABLES"，但函数体没用这个参数。ALLOWED_TABLES 看着生效实际形同虚设。

**规则**：加默认参数必须同步实现 + 补测试。已在 `CLAUDE.md` Lessons Learned 记录。

### 2. 全局 monkey-patch 必须还原

**场景**：测试里 `m.llm = fake` 改了模块属性，如果不在 finally 里还原，会污染后续测试（test_llm_registry 拿到 fake 而不是真 model）。

**规则**：测试 fixture 改全局状态必须用 try/finally 或 monkeypatch 自动还原。

### 3. 测试跟生产代码要同步演进

**场景**：generate_intent 改用 `get_llm("generate_intent")` 后，测试还用老的 `m.llm = fake` patch，3 个 e2e 测试从此失败（fake 没生效，走真远端超时）。半年没人发现。

**规则**：重构"模块级全局变量 → 函数查询"时，测试的 mock 方式必须同步改。

### 4. asyncio.gather 不要无脑吞所有异常

**场景**：`return_exceptions=True` 把 CancelledError 也吞了，客户端断开时 recall 还在跑完所有关键词，浪费资源。

**规则**：吞业务异常（单个关键词失败不影响其他），但 CancelledError 必须向上传播。

### 5. 文档要写权衡，不只写"做了什么"

**场景**：`cached_pre_state` 保留 cache 有性能收益但retry 时可能不匹配新 plan 拓扑。如果只写"我们做了 cache"而不写"已知风险"，下一个改这里的人会踩坑。

**规则**：所有非显然的设计决策都要在代码注释和文档里写清楚"为什么这么做 + 已知风险 + 未来怎么演进"。
