## 关于安全渗透测试的讨论

**藤子的疑问**：安全渗透测试有必要吗？目标是 AI 应用/AI 数据岗位，渗透应该是网络安全专业的活。

**结论**：不需要。AI 岗位的测试重点应是：
1. ✅ 单元测试（证明代码逻辑正确）
2. ✅ 集成测试（证明组件协作正常）
3. ❌ 渗透测试（安全专业的活，跨领域了）

面试官也更看重你能不能写测试、能否用 ruff/pytest 规范代码，而不是会不会渗透。

### 4. 测试中发现的 Bug 并修复

**Bug**：`SELECT 'DROP' AS label` 被误判为危险操作

**原因**：`\bDROP\b` 无法区分关键字是否在引号内

**修复**：在关键字匹配前，先用正则 `r"'[^']*'"` 移除所有字符串字面量，再对剩余部分做关键字匹配

```python
# 修复前（会误杀）
if re.search(pattern, sql_upper):    # 引号内的 DROP 也会被匹配

# 修复后（不会误杀）
sql_no_strings = re.sub(r"'[^']*'", "", sql_upper)  # 先删掉引号内容
if re.search(pattern, sql_no_strings):               # 再匹配关键字
```
